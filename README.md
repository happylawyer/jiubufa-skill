# 要件审判九步法 AI 技能 v4.0

基于邹碧华《要件审判九步法》的 AI 裁判辅助工具。输入案件卷宗材料，AI 严格按照九步法流程进行请求权基础分析、举证责任分配、事实认定与要件涵摄，最终生成裁判文书草稿。

## 核心特性

- **九步法全流程**：固定权利请求 → 请求权基础 → 抗辩权基础 → 构成要件分析 → 主张检索 → 争点整理 → 举证责任分配 → 事实认定 → 要件归入
- **双模式**：极速全自动（一键出稿）/ 专家步步确认（逐步审核）
- **六条铁律**：禁止越权补事实、禁止和稀泥、禁止跳步、禁止旧法套用、禁止编造法学理论、必须回应核心抗辩
- **法条风控**：自动校验法律时效性、条文准确性、法学理论真实性
- **"八个一致"校验**：文书输出前强制自我审计
- **材料缺失预处理**：自动评估卷宗完整性，标注受影响步骤
- **被告缺席处理**：自动适配被告未答辩/缺席场景

## 适用场景

- 法官/法官助理起草裁判文书
- 律师预判案件走向
- 法学教学中的案例分析训练

## 快速开始

### Claude Code（推荐）

```bash
# 方法一：直接安装
mkdir -p ~/.claude/skills/jiubufa
cp claude-code/SKILL.md ~/.claude/skills/jiubufa/SKILL.md

# 方法二：符号链接（方便更新）
git clone https://github.com/happylawyer/jiubufa-skill.git ~/jiubufa-skill
mkdir -p ~/.claude/skills/jiubufa
ln -sf ~/jiubufa-skill/claude-code/SKILL.md ~/.claude/skills/jiubufa/SKILL.md
```

安装后在 Claude Code 中输入 `/jiubufa` 即可启动。

### ChatGPT / GPTs

1. 打开 [ChatGPT](https://chat.openai.com)，进入「设置 → 自定义指令」或创建 GPT
2. 将 `prompt/system-prompt.md` 的全部内容粘贴到「系统提示词」中
3. 保存后，在对话中提供案件材料即可开始分析

### Claude (Web / API)

1. 在 [Claude](https://claude.ai) 对话中，将 `prompt/system-prompt.md` 的内容作为第一条消息发送
2. 或在 API 调用中设为 `system` 参数
3. 然后提供案件材料开始分析

### 通义千问 / 文心一言 / Kimi / DeepSeek 等国产大模型

1. 将 `prompt/system-prompt.md` 的内容粘贴到「系统提示词」或「角色设定」中
2. 提供案件材料即可

### Codex CLI

```bash
# 将提示词设为 Codex 的系统指令
codex --system-prompt "$(cat prompt/system-prompt.md)" "请分析以下案件材料..."
```

### 微信 AI 机器人（ClawBot 等）

将 `prompt/system-prompt.md` 内容配置为机器人的系统提示词即可。

## 使用示例

```
用户：[提供起诉状、答辩状等材料]，请选择模式 1

AI：
├── ⚠️ 材料缺失预处理报告
├── 第1步：固定权利请求
├── 第2步：确定权利请求基础规范（含法条风控记录）
├── 第3步：确定抗辩权基础规范
├── 第4步：构成要件分析
├── 第5步：诉讼主张检索
├── 第6步：争点整理
├── 第7步：举证责任分配
├── 第8步：事实认定
├── 第9步：要件归入
├── "八个一致"校验
└── 裁判文书草稿（事实查明 + 本院认为 + 判决主文）
```

## 注意事项

- 本工具生成的是**辅助草稿**，不能替代法官的独立裁判。所有输出必须经过法官审核。
- AI 可能存在法条记忆偏差，工具内置了法条风控机制，但仍建议核对法条原文。
- 仅适用于中国大陆民商事案件。
